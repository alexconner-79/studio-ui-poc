# v0.8.5 -- Shapes, Drawing & Promote Workflow

**Status:** Complete
**Previous:** [v0.8.0](v0.8.0.md) | **Next:** [v0.9.0](v0.9.0.md)

## Summary

Adds shape primitives (Rectangle, Ellipse, Line, Frame) and toolbar creation tools so users can draw directly on the canvas in Design Mode. Introduces the **Promote-to-Build workflow** -- the key differentiator that bridges design freedom with production code quality. Also absorbs the previously planned Image Handling work, treating images as rectangles with image fills.

---

## Changelog

### 0.8.5.1 -- Shape Node Types

- [x] Add to `BuiltInNodeType` in `compiler/types.ts` and `apps/web/lib/studio/types.ts`:
  - `Rectangle` -- renders as `<div>` with configurable fill, stroke, radius. Default `compile: false`.
  - `Ellipse` -- renders as `<div>` with `border-radius: 50%`. Default `compile: false`.
  - `Line` -- renders as styled `<hr>` or SVG `<line>`. Default `compile: false`.
  - `Frame` -- like Box but with auto-layout toggle, clip, constraints. Default `compile: true` (Frames are structural).
- [x] Add renderer cases in `renderer.tsx` for all four types
- [x] Add property panel sections for shape-specific properties (fill, stroke, corner radius)
- [x] Shape types appear in the component palette under a "Shapes" category

**Key files:** `compiler/types.ts`, `apps/web/lib/studio/types.ts`, `apps/web/components/studio/renderer.tsx`, `apps/web/components/studio/property-panel.tsx`, `apps/web/lib/studio/node-schemas.ts`

### 0.8.5.2 -- Toolbar Creation Tools

- [x] Add `currentTool` state to the editor store: `"select" | "frame" | "rectangle" | "ellipse" | "text" | "line"`
- [x] Tool buttons in the top bar (Design Mode only, except Select which works in both):
  - **Select** (V) -- click to select, drag to move
  - **Frame** (F) -- click+drag to create a Frame at drawn coordinates
  - **Rectangle** (R) -- click+drag to draw a Rectangle
  - **Ellipse** (O) -- click+drag to draw an Ellipse
  - **Text** (T) -- click to place a Text node, immediately enters inline edit
  - **Line** (L) -- click+drag to draw a Line
- [x] Canvas pointer-event layer intercepts events when a creation tool is active
- [x] On click+drag: create the node at pointer position with drawn dimensions via `addNode`
- [x] After creation: automatically switch back to Select tool and select the new node
- [x] Keyboard shortcuts work globally (not just when toolbar is focused)

**Key files:** `apps/web/lib/studio/store.ts`, `apps/web/components/studio/editor-layout.tsx`, `apps/web/components/studio/editor-canvas.tsx`

### 0.8.5.3 -- Promote-to-Build Workflow

- [x] **Right-click context menu actions:**
  - "Promote to Build" -- sets `compile: true`. If node uses absolute positioning, offers to wrap in an auto-layout Frame first.
  - "Convert to Component" -- wraps a shape/group in a semantic node type (user picks: Card, Section, Box, Button, etc.). Sets `compile: true`.
  - "Move to Design" -- sets `compile: false`.
- [x] **Automatic suggestions (non-blocking toasts):**
  - Rectangle with text child → "Convert to Button?"
  - Rectangle with image fill + text → "Convert to Card?"
  - Group of evenly-spaced elements promoted → "Wrap in Stack with auto-calculated gap?"
- [x] **Structure linter (pre-compile check):**
  - Scan Build Mode tree for: hardcoded pixel positions that could be auto-layout, deep nesting that could be flattened, raw divs where semantic types fit better
  - Show warnings in a "Compile Health" panel (non-blocking)
  - Each warning has a one-click fix action

**Key files:** `apps/web/components/studio/context-menu.tsx`, `apps/web/components/studio/editor-layout.tsx`, `apps/web/lib/studio/structure-linter.ts` (new)

### 0.8.5.4 -- Compiler Guardrails

- [x] Compiler skips `compile === false` subtrees (implemented in v0.8.0, verified here)
- [x] New emitter cases in all emitters (Next.js, Vue, Svelte, HTML, Expo):
  - `Frame` → `<div>` with `position: relative`, `overflow: hidden` (or flex if auto-layout enabled)
  - `Rectangle` → styled `<div>` (only if `compile: true`)
  - `Ellipse` → styled `<div>` with `border-radius: 50%` (only if `compile: true`)
  - `Line` → `<hr>` or `<div>` with height/border (only if `compile: true`)
- [x] Token snapping at compile time: hardcoded pixel values in promoted nodes rounded to nearest design token within configurable threshold

**Key files:** `compiler/emitters/nextjs.ts`, `compiler/emitters/vue.ts`, `compiler/emitters/svelte.ts`, `compiler/emitters/html.ts`, `compiler/emitters/expo.ts`

## Test Plan

### Prerequisites

- Dev server running
- v0.8.0 features complete (dual-mode canvas, resize, move, guides)
- At least one project with a screen

### Tests

**1. Shape Creation (Drawing Tools)**
1. Switch to Design Mode
2. Press R -- verify Rectangle tool is active (toolbar highlights)
3. Click+drag on the canvas -- verify a Rectangle is created at the drawn position/size
4. Verify the Rectangle has `compile: false` by default (dashed border, dimmed in layers)
5. Press O -- draw an Ellipse. Verify it renders as a circle/ellipse
6. Press L -- draw a Line. Verify it renders as a line
7. Press F -- draw a Frame. Verify it has `compile: true` by default
8. Press T -- click on canvas. Verify a Text node is created and inline editing starts
9. Press V -- verify Select tool is active and drawing tools are disabled

**2. Promote-to-Build Workflow**
1. Create a Rectangle in Design Mode
2. Right-click > "Promote to Build" -- verify `compile: true`, dashed border removed
3. Compile -- verify the Rectangle appears in the generated output as a styled `<div>`
4. Create another Rectangle, add a Text child inside it
5. Promote the Rectangle -- verify a toast suggests "Convert to Button?"
6. Accept the suggestion -- verify the node type changes to Button
7. Right-click a Build Mode node > "Move to Design" -- verify it becomes `compile: false`

**3. Convert to Component**
1. Draw a Rectangle with text and an icon inside
2. Right-click > "Convert to Component" -- verify a picker appears with semantic types
3. Select "Card" -- verify the node is wrapped in a Card with `compile: true`
4. Verify the Card appears correctly in the layer tree

**4. Structure Linter**
1. Promote several absolutely-positioned nodes to Build Mode
2. Open the "Compile Health" panel
3. Verify warnings appear for hardcoded pixel positions
4. Click a warning's fix action -- verify the node is wrapped in an auto-layout Frame

**5. Image Handling**
1. Add an Image node to a screen
2. Set a `src` URL -- verify the image appears on canvas
3. Change `objectFit` to "cover" -- verify the canvas updates
4. Set `aspectRatio` to "16:9" -- verify dimensions constrain
5. Open the mask section, select "Circle" -- verify the image clips to a circle
6. Compile -- verify `object-fit`, `aspect-ratio`, and `clip-path` appear in output

**6. Shape Properties**
1. Select a Rectangle
2. Change fill colour -- verify canvas updates
3. Add a stroke (2px, red) -- verify it renders
4. Set corner radius to 12px -- verify rounded corners
5. Set rotation to 45 degrees -- verify the shape rotates on canvas
6. Compile (after promoting) -- verify all properties appear in output

**7. Compiler Guardrails**
1. Create a mix of Design Mode and Build Mode nodes
2. Compile -- verify only `compile: true` nodes appear in output
3. Verify no empty containers or orphaned styles from skipped nodes
4. Check that token snapping rounds pixel values to nearest design token in output

**8. Keyboard Shortcuts**
1. Press V, F, R, O, T, L -- verify each tool activates
2. Switch to Build Mode -- verify drawing tools (F, R, O, T, L) are disabled or switch to Select
3. Press Escape while a tool is active -- verify it switches back to Select

## Bug Fixes

- **Shape drawing coordinates corrected for zoom**: All shape tools (Rectangle, Ellipse, Line, Frame, Text click) now correctly divide raw viewport coordinates by the canvas scale factor before creating nodes. Shapes now appear at the correct position and size at any zoom level.
- **R shortcut unblocked**: The `R` key now correctly activates the Rectangle tool. The rename action was remapped to `F2` (see v0.8.0 bug fixes).

## Deferred

- Shape tool UX polish (size label during draw, min draw threshold, line Shift-snap to 45°) tracked in [v0.8.9.md](./v0.8.9.md)

## Known Limitations

- Gradient editor is basic (two-stop linear/radial only); advanced gradient editing deferred to v0.10.0
- Token snapping uses a fixed threshold; may need tuning per project
- Structure linter suggestions are heuristic-based and may not always be correct
- Image cropping (server-side via sharp) deferred to a later release
- Shape rotation does not work in the Expo emitter (React Native transform limitations)
