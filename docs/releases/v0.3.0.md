# v0.3.0 -- Design Systems

**Status:** Complete
**Previous:** [v0.2.0](v0.2.0.md) | **Next:** [v0.4.0](v0.4.0.md)

## Summary

Introduced real design system thinking into the tool. Users can create reusable components, import design tokens from industry formats, scaffold design systems from scratch, edit themes visually, run accessibility audits, and start from curated templates.

## Changelog

### Custom Composite Components

- [x] Select nodes on canvas, right-click "Create Component"
- [x] Saves subtree as `.component.json` in `components/` directory
- [x] Appears in palette under "Custom" category
- [x] Dragging creates an instance (deep copy with fresh IDs)

### Starter Component Library

- [x] Hero Section, Feature Grid, Pricing Table, Contact Form
- [x] Navigation Header, Footer, Login/Signup Form
- [x] Stored as `.component.json` in `templates/` directory
- [x] Available from "Templates" tab in palette

### Design System Import

- [x] W3C Design Tokens format -- full community group spec
- [x] Tailwind config import -- parse `tailwind.config.js`, extract theme
- [x] shadcn theme import -- CSS variables as color tokens

### Design System Scaffolding

- [x] "Create a Design System" wizard
- [x] Pick base palette or enter brand colours
- [x] Choose typography and spacing scales
- [x] Generates `design-tokens.json` + `tailwind.config.ts` + CSS variables
- [x] Applies immediately to editor and compiler

### Theme Editor

- [x] Visual colour/typography/spacing picker, updates tokens live
- [x] Preview changes across all screens in real time

### Accessibility Checker

- [x] Colour contrast, missing alt text, heading order gaps, interactive labelling
- [x] Builds on `compiler/lint.ts` infrastructure
- [x] Inline canvas warning indicators

### Template Gallery

- [x] SaaS landing page, admin dashboard, blog, e-commerce product page
- [x] "Start from template" option on screen list page

## Known Issues at Release
