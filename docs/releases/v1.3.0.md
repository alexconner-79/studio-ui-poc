# v1.3.0 -- Advanced Vector Tools

**Status:** Future
**Previous:** [v1.2.0](v1.2.0.md) | **Next:** [v1.4.0](v1.4.0.md)

## Summary

Professional vector editing capabilities for Design Mode. Adds boolean shape operations, a pen tool for custom paths, and blend modes. These are power-user visual tools deferred from v0.10.7 -- they serve the "designing shadows to perfection" end of the design craft spectrum and belong after the core workflow is established and stable.

---

## Planned Features

### Boolean Operations

- [ ] Available on shape primitives (Rectangle, Ellipse, Path): union, subtract, intersect, exclude
- [ ] Produces a compound `BooleanGroup` node containing the source shapes and the operation type
- [ ] Canvas renders the boolean result using SVG path operations (via paper.js or clipper.js)
- [ ] Result can be "flattened" into a single `Path` node
- [ ] Property panel shows boolean operation type with option to change or flatten
- [ ] New node type: `BooleanGroup` (default `compile: false`)
- [ ] Compiler: if promoted, emits as an inline SVG

**Key files:** `apps/web/lib/studio/types.ts`, `compiler/types.ts`, `apps/web/lib/studio/boolean-ops.ts` (new), `apps/web/components/studio/renderer.tsx`

### Pen Tool (Vector Editing)

- [ ] New tool in toolbar: **Pen** (keyboard shortcut: P)
- [ ] Click to place anchor points; drag to create bezier curves
- [ ] Edit mode: double-click a Path node to enter vector editing, showing anchor points and handles
- [ ] Anchor point types: corner (sharp), smooth (symmetric handles), asymmetric
- [ ] Close path: click first anchor point to close the shape
- [ ] New node type: `Path` -- stores SVG path data (`d` attribute string)
- [ ] Default `compile: false` (design-only unless promoted to Build Mode)
- [ ] Compiler: if promoted, emits as `<svg><path d="..."/></svg>`
- [ ] Property panel: fill, stroke, and path-specific controls (anchor count, open/closed)

**Key files:** `apps/web/lib/studio/store.ts`, `apps/web/components/studio/editor-canvas.tsx`, `apps/web/components/studio/renderer.tsx`, `compiler/types.ts`, `compiler/emitters/*.ts`

### Blend Modes

- [ ] Add `blendMode` to `NodeStyle`: normal, multiply, screen, overlay, darken, lighten, colour-dodge, colour-burn, hard-light, soft-light, difference, exclusion
- [ ] Dropdown in property panel (under Effects section)
- [ ] Compiler: `mix-blend-mode` CSS property
- [ ] Works on any node type

**Key files:** `apps/web/lib/studio/types.ts`, `compiler/types.ts`, `apps/web/components/studio/style-sections.tsx`, `compiler/emitters/*.ts`

## Test Plan

**1. Boolean Operations**
1. Create two overlapping shapes (Rectangle + Ellipse)
2. Select both, apply "Subtract" -- verify ellipse cuts into the rectangle
3. Change to "Union" -- verify they merge
4. Change to "Intersect" -- verify only the overlap remains
5. "Flatten" the result -- verify it becomes a single Path node
6. Promote and compile -- verify an inline SVG is emitted

**2. Pen Tool**
1. Press P to activate the pen tool
2. Click to place anchor points forming a triangle -- verify the path renders
3. Drag while placing to create curves -- verify bezier handles appear
4. Close the path by clicking the first anchor -- verify it fills
5. Double-click the path to enter edit mode -- verify anchor points are visible and draggable
6. Promote and compile -- verify SVG path output

**3. Blend Modes**
1. Stack two overlapping coloured shapes
2. Set the top shape's blend mode to "multiply" -- verify the blending effect
3. Change to "screen" -- verify the effect changes
4. Compile -- verify `mix-blend-mode` CSS property in output
