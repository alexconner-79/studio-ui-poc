## Test Plan

### Prerequisites

- Dev server running
- Multiple screens in a project (for compiler scaling tests)
- AI provider configured (for AI config tests, optional)

### Tests

**1. Toast Notifications**
1. Trigger a save with the dev server's backend stopped
2. Verify a toast notification appears (not just a console error)
3. Navigate to `/studio`, simulate a failed API call
4. Verify toast appears instead of silent failure
5. Verify no `alert()` calls remain (check project creation error path)

**2. Path Resolution**
1. Verify `studio.config.json` contains path configuration
2. Move the screens directory to a non-default location, update config
3. Verify the API still finds and serves screens correctly
4. Verify compile still works with the updated paths

**3. Incremental Compilation**
1. Create a project with 5+ screens
2. Save one screen -- verify only that screen's generated file is updated
3. Check `.compile-cache.json` exists and tracks mtimes
4. Save without changes -- verify no files are rewritten (check file mtimes)
5. Introduce a validation error in one screen -- verify other screens still compile

**4. Async Compilation**
1. Save a screen
2. Verify the browser remains responsive during compilation (no frozen UI)
3. Verify compilation completes and the generated file updates

**5. Lucide Icons**
1. Stop the dev server, clear Turbopack cache
2. Start the dev server -- verify initial compile is under 30 seconds (down from ~2 min)
3. Open the icon picker -- verify search still works
4. Add an Icon node with a specific icon -- verify it renders correctly on canvas
5. Save and compile -- verify the correct `import { IconName } from "lucide-react"` is emitted

**6. ESLint Clean**
1. Run `pnpm lint` from `apps/web`
2. Verify no eslint-disable comments remain in the listed files
3. Verify no new lint errors are introduced

## Known Limitations

- **Incremental compilation cache is file-based; in Supabase mode, spec changes bypass the filesystem.** *Permanent constraint for Supabase mode -- in-memory compilation is fast enough that caching is not needed. Not planned for change.*
- **AI config consolidation may require migration for existing `studio.config.json` files.** *Addressed in v0.9.4 -- migration handled by merging new keys into existing config on first read.*
- **`next/image` with `fill` prop on the canvas Image node requires the parent element to have `position: relative` and an explicit height.** *Canvas renderer wraps the image in a positioned container. Acceptable constraint for the canvas preview context.*
