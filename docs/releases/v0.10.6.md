# v0.10.6 — Canvas Design Fundamentals

**Status:** Complete
**Previous:** [v0.10.5](v0.10.5.md) | **Next:** [v0.10.7](v0.10.7.md)

## Summary

The canvas has the structural foundation — Design/Build mode, resize handles, drag-to-move, auto-layout, component instances — but day-to-day design work is not viable yet. Typography cannot be controlled beyond a single text field. Spacing cannot be set per-side. There are no keyboard shortcuts for common operations, no grouping, and the properties panel is missing the controls designers reach for constantly.

This release fixes the three blockers identified as preventing real design work:

- **Canvas operations** — keyboard shortcuts for every common action, grouping, align & distribute
- **Typography** — full font control in the properties panel: family, size, weight, line height, letter spacing, alignment
- **Spacing & layout** — per-side padding and gap controls, accessible directly from the panel without writing raw CSS

After this release, a designer can open Studio and do real layout and typography work without hitting a wall every few seconds.

---

## Why This Matters

A tool that requires workarounds for basic operations like "nudge this element 4px" or "make this text 20px semibold" is not a design tool — it is a prototype. The goal of this release is to close that gap completely so every fundamental design operation works the way an experienced Figma or Sketch user would expect.

**Group navigation:**
- [x] `Cmd+Enter` — enter selected group/frame to select first child

**Shortcut reference:**

### 0.10.6.4 — Typography Controls

The properties panel gains a dedicated Typography section whenever a text node or a node with text content is selected. All controls apply immediately on the canvas via inline style updates — no save required.

- [x] **Font family** — searchable dropdown populated from system fonts + any fonts declared in the active DS token set; shows a live preview of each family
- [x] **Font size** — numeric input with `px` unit; up/down arrow keys increment/decrement by 1; `Shift+arrow` increments by 10
- [x] **Font weight** — dropdown showing available weights for the selected family: Thin (100), ExtraLight (200), Light (300), Regular (400), Medium (500), SemiBold (600), Bold (700), ExtraBold (800), Black (900)
- [x] **Line height** — numeric input; supports unitless multiplier (e.g. `1.5`) or `px` value; defaults to `normal`
- [x] **Letter spacing** — numeric input in `em` or `px`; supports negative values
- [x] **Text alignment** — icon button group: left, centre, right, justify
- [x] **Text decoration** — icon toggles: underline, strikethrough
- [x] **Text transform** — icon toggles: none, uppercase, lowercase, capitalise
- [x] **Colour** — colour swatch that opens an inline colour picker; supports token type-ahead (type `accent` → suggests `color.accent = #7c3aed`)

## Design Decisions

**Why bracket shortcuts (`Cmd+[`/`Cmd+]`) for layer ordering?** Matches Figma and Sketch conventions that experienced designers already know. `Cmd+Up/Down` conflicts with browser text navigation shortcuts.

**Why a floating toolbar for align/distribute?** These operations only make sense with multiple nodes selected. A contextual floating toolbar that appears and disappears is less cluttered than a permanent panel section that is greyed out most of the time.

**Why font family shows system fonts?** The DS token system handles curated brand fonts. The system font list is a fallback for designers who want to try fonts before committing them to the token set. Only fonts declared in DS tokens become token-bound; system font picks are stored as raw values.

**Why expose margin separately from padding?** Padding is a property of the node itself. Margin is a relationship between the node and its parent's layout. Keeping them in distinct panel sections avoids the confusion of which property affects what.

### 0.10.6.8 — Property Panel Clarity & Override Visibility

Resolved a structural confusion in the property panel where the same visual concerns were controllable in two places with no clear hierarchy. Established a clear two-section model and added a visual indicator for nodes with style overrides.

**Panel structure:**
- [x] **"Component" section label** added above the component props block — makes it immediately clear that `text`, `level`, `variant`, etc. are semantic/structural props, not CSS overrides
- [x] **`fontFamily` removed** from `Heading` and `Text` node schemas — font family is a CSS fine-tune; it belongs only in the Typography style overrides section below

**Promoted to Component section:**
- [x] **Text Align** (`Left` / `Centre` / `Right` / `Justify`) moved from the Typography overrides section into Component props for `Heading` and `Text` — alignment is a content decision, not a CSS override; one place only
- [x] **Text Align removed** from the Typography style overrides section entirely
- [x] **Text Colour** added to Component props as a DS colour token selector — shows all colour tokens from the active design system with hex swatches; includes a custom hex/rgb fallback. Also remains in the Typography section as a raw CSS override for off-DS colours

**Layers panel:**
- [x] **Override dot** — a small accent-coloured dot appears after the layer name for any node that has inline CSS style overrides applied (`node.style` is non-empty). Lets designers scan the layers tree at a glance to see which nodes have been manually styled beyond the component's DS defaults — analogous to Figma's override indicator

### 0.10.6.9 — Property Panel Contextual Redesign

The right panel now shows only what is relevant to the selected element. Sections that don't apply to a node type are completely absent — not collapsed, not greyed out.

- [x] **Section order** — Identity → Position & Size → Layout → Typography → Fill/Border → Effects → Interactions → Data
- [x] **Position & Size widget** — compact X / Y / W / H grid near the top of the panel for every node. Artboard (root) shows W/H only (no X/Y)
- [x] **Contextual section visibility** by node type:
  - Frame / Stack / Grid / Section / ScrollArea / Card / Form / Modal / Nav / Tabs — Layout + Fill + Border + Effects + Interactions
  - Heading / Text — Component + Typography + Border + Effects + Interactions (no Fill — text colour is in Typography)
  - Button / Link — Component + Fill + Border + Effects + Interactions
  - Input — Component + Typography + Fill + Border + Effects + Interactions
  - Icon — Component + Effects + Interactions
  - Image — Size + Border + Effects only (no Fill, no Typography, no Interactions)
  - Divider — Size + Fill (line colour) only
  - Spacer — Size only
  - ComponentInstance — Component (DS badge + variants/props) + Fill + Border + Effects + Interactions
- [x] **Layout section** moved to top of style sections (was last)
- [x] **Interactions** moved to after style sections (was before them — wrong order)
- [x] **Data binding** moved to after Interactions (was before styles — wrong order)
- [x] **"Overrides" wrapper** removed — each section is self-labelled
- [x] **Delete button** removed from panel — delete via Backspace/Delete key on canvas
- [x] **Section expand/collapse chevron** replaced with SVG chevron matching the left panel (right-pointing, rotates 90° when open), positioned to the right of the section title

**Key files:** `apps/web/components/studio/property-panel.tsx`, `apps/web/components/studio/style-sections.tsx`

## Known Limitations

- **Arrow-key nudge in Build Mode** — Permanent constraint. Build Mode nodes are flow-positioned; nudging changes `margin` or `order`. Design Mode (absolute-positioned) nodes nudge predictably.
- **Align & Distribute for cross-container nodes** — Permanent constraint for Build Mode. Alignment uses canvas coordinate space (correct for Design Mode). Mixed-parent alignment in Build Mode may produce unexpected results; a UI note is shown when detected.
- **Full token picker (categorised, searchable)** — Addressed in v0.10.8 Component Editability. The type-ahead dropdown (8 matches) is sufficient for v0.10.6. See BACKLOG.md — Full token picker.
- **Copy/paste across screens** — See BACKLOG.md — Copy/paste across screens.
