# v0.8.9 — Canvas Interaction UX Redesign

**Status:** Complete
**Theme:** Elevate the fidelity and discoverability of all canvas interaction patterns introduced in v0.8.0–v0.8.5

---

## Context

The features shipped in v0.8.0 (Design Mode, Build Mode, resize/drag/smart guides) and v0.8.5 (Shape drawing tools) are functionally present but have significant UX debt. Users find the interactions hard to discover, unclear in feedback, and difficult to control precisely. This release is a dedicated second pass to make those interactions feel polished and professional.

---

## Planned Work

### 0.8.9.1 — Resize Handle UX ✓

- [x] Replace hairline corner dots with clearly visible grab handles (8-point: corners + midpoints) — `CORNER_SIZE` increased to 9, border upgraded to 2px, stronger shadow, hover scale 1.25×
- [x] Handles appear only on hover/selection — not permanently visible to reduce noise — handles already only render when a node is selected; `hoveredHandle` state drives scale/shadow on hover
- [x] Handles show a tooltip with current W × H on hover — dark tooltip rendered above the node when any corner handle is hovered
- [x] Constrained (Shift) and center-origin (Alt) resize modes should show a visual indicator — `resizeModifier` state renders "⇧ Proportional" / "⌥ From centre" pill below dimension label during active resize
- [x] Minimum drag threshold (e.g. 4px) before resize activates — prevents accidental resizes from clicks — `activated` flag added to `resizeRef`; `onMove` exits early until 4px displacement

### 0.8.9.2 — Drag-to-Move Clarity ✓

- [x] Show a move cursor (✥) when hovering a selected node in Design Mode — `cursor-move` now only applies to `isAbsoluteInDesign && isSelected`; unselected absolute nodes show `cursor-pointer` to indicate "click to select first"
- [x] Ghost/drop shadow should follow the cursor during drag — `isFreeDragging` now sets 60% opacity + `0 8px 24px` drop shadow + accent outline, making the node clearly feel "lifted"; distinct from the DnD kit drag which keeps 30% opacity
- [x] Display `X: N  Y: N` live coordinates near the node while dragging — dark pill above the node centre; format changed from `x, y` to `X: N  Y: N` with monospace font
- [x] Snap to grid / smart guides should also snap during move — already implemented via `snapMove()` in `handleFreePointerDown`; confirmed present
- [x] Distinguish between "drag node" and "drag handle" interactions clearly — cursor difference (move vs resize cursors) already existed; drag lift effect now makes it visually obvious the node is being moved

### 0.8.9.3 — Smart Guide Discoverability ✓

- [x] Smart guides currently only appear at precise alignment moments — snap threshold increased from 4px to 8px in `geometry.ts`; guides trigger much more naturally during drag/resize
- [x] Add a "Snap" toggle in the top toolbar — magnet icon `SToolbarButton` added next to undo/redo; reads `snapEnabled` from store, calls `toggleSnap()`; when disabled, snap is bypassed in both drag-to-move (`renderer.tsx`) and resize (`spacing-overlay.tsx`) and all active guides are cleared
- [ ] Edge spacing indicators when holding Alt while hovering — deferred; requires per-node Alt+hover detection across the whole tree, significant complexity. Noted for v1.4.0 UX pass.
- [x] Add a brief flash animation when snap occurs — `GuideLineEl` now runs a `snap-flash` keyframe (opacity 1→0.3→1→0.85 over 350ms) each time a guide is rendered; guide lines also upgraded from dashed to solid 1.5px for better visibility

### 0.8.9.4 — `compile: false` Visual Indicators

- [x] Design-only nodes (compile: false) should have a clearly distinct visual treatment on the canvas:
  - Upgraded to a 1.5px amber dashed `outline` (was 1px faint text-tertiary border)
  - Persistent amber "Design only" pill badge at top-right (always visible, not just when selected). When selected, the existing selection chip shows "Design" instead. (`renderer.tsx`)
  - Diagonal stripe overlay: stripe thickness increased from 4→5px, opacity 0.08→0.10 — subtle but detectable on white backgrounds
- [x] The badge is hidden when the node's px width or height is < 40px (a small amber corner dot is shown instead to avoid clutter)
- [x] All indicators (`outline`, stripes, badge) are suppressed when `previewMode` is true

### 0.8.9.5 — "Promote to Build" / "Move to Design" UX Planning

- [x] Canonical compile state change: context menu ("Promote to Build" / "Move to Design") AND right panel header toggle — both already existed; confirmed as the correct dual-entry model
- [ ] "Promote to Build" confirmation when children are also compile: false — deferred; implications of cascading vs partial promotion need more design thinking. Noted for v1.4.0 UX pass.
- [ ] "Move to Design" warning when node is referenced in compiled output — deferred to v1.4.0
- [x] Promote suggestion toast: fires after drawing a Rectangle, Ellipse, Line, or Text node in Design Mode — "Shape/Text added as Design-only" with an inline "Promote →" action button (6s duration). Implemented in `editor-canvas.tsx` using Sonner action toast.
- [x] Build / Design-only node count status bar added at the bottom of the left panel — shows "X Build · Y Design-only" in accent/amber colours. Recursively counts all nodes in `spec.tree`. Hidden if no design-only nodes exist. (`editor-layout.tsx` — `NodeCountStatus` component)

### 0.8.9.6 — Topbar Preview Controls Review

- [x] "Canvas Preview" chip removed entirely — users don't need to see the artboard again without editor chrome; it was a source of confusion
- [x] "Open Page" (external-link icon) removed — replaced by a single **"Preview"** chip (playback icon) in the topbar. Clicking navigates to a new Framer-style preview route within the app.
- [x] "Open Page" / "Preview" button is truly disabled when nothing has been compiled — `pointer-events: none`, opacity 0.45, tooltip: "Compile first (Cmd+S) to preview the page". Button only activates after a successful `Cmd+S` save + compile.
- [x] Framer-style preview route created at `apps/web/app/preview/page.tsx` — fixed Studio topbar (logo, screen name, device toggles Mobile/Tablet/Desktop, "← Back to Editor" link) with an iframe rendering the compiled page below. Device toggles control the iframe container width (390px / 768px / full-width) with a card shadow for narrow modes.
- [x] `Cmd+P` now navigates to the preview route (if compiled) or shows a toast "Compile first (Cmd+S) to preview the page" (if not).

### 0.8.9.7 — Shape Tool UX Polish ✓

- [x] After drawing a shape, automatically activate the Select tool — confirmed present for both shapes and text; `setCurrentTool("select")` called in `handlePointerUp` and text-click handler in `editor-canvas.tsx`
- [x] Show a size label (`W × H`) near the cursor while drawing — dark monospace pill rendered at `(curX + 10, curY + 10)` while drag exceeds 4px; shows `W × H` for rect/ellipse/frame, `Npx` length for line; disappears on pointer-up
- [x] Minimum draw size: if user draws < 4px, treat as a click and insert a default-sized shape centred on click point — `MIN_DRAW_PX = 4` constant; default sizes: Frame 200×200, Rectangle/Ellipse 100×100, Line 100px; centred on click point
- [x] Line tool: snap to 45° increments when holding Shift — `handlePointerMove` checks `currentTool === "line" && e.shiftKey`; endpoint snapped to nearest 45° using `atan2` → `Math.round(angle / (π/4)) * (π/4)`

## Known Limitations

| # | Limitation | Disposition |
|---|-----------|-------------|
| 1 | Smart guides only work for direct siblings in the same container | Permanent constraint — cross-container guides are a major engine investment, deferred to v1.3+. |
| 2 | Drag-to-move is only available for absolute-positioned nodes in Design Mode | Permanent constraint — Build Mode nodes follow DOM flow, not free positioning. By design. |
| 3 | Resize handles may jitter at very high zoom levels (> 300%) | Addressed in v1.4.0 UX Refinement Pass. |
