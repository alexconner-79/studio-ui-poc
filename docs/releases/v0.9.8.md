# v0.9.8 -- Onboarding & Editor Chrome

**Status:** Complete
**Previous:** [v0.9.5](v0.9.5.md) | **Next:** [v0.10.0](v0.10.0.md)

## Summary

Fixes the structural chrome and new user flow before layering Design System features on top. The editor's left panel, right panel, and screens navigation are all structurally incomplete. The project creation flow doesn't explain GitHub connection or make project type choices clear. This release addresses all of that so v0.10.0 DS Authoring lands on a solid foundation.

## Known Limitations

- **Artboard tabs are per-session**: Tab state lives in Zustand and resets on page refresh. Persisting open tabs to Supabase deferred to v0.10.0. Not a blocking UX issue -- screens are quick to re-open via the Screens panel.
- **GitHub OAuth not implemented**: Only PAT (Personal Access Token) entry in v0.9.8. Full OAuth flow requires a registered GitHub App and a server-side callback endpoint. Planned for v0.10.0 alongside the brownfield DS import feature.
- **GitHub PAT stored in localStorage**: Per-project, not synced across devices. Migration to Supabase `project_settings` table in v0.10.0. For now, acceptable -- PAT is a per-device setup step.
- **Onboarding wizard DS step is preview-only**: Choosing "Connect your component library" in the wizard does nothing until v0.10.0 implements the actual import flow. The wizard correctly labels this as deferred.
- **Right panel improvements are a first pass**: Section order restructure and Common/Advanced disclosure are in v1.4.0. The first pass (compile toggle, contextual sections, no-selection state) is implemented.
- **Unsaved-changes dot on artboard tabs**: Would require per-tab dirty state tracking (currently dirty is global per screen). Planned for v1.4.0 alongside the full tab UX polish pass.
