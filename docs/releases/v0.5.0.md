# v0.5.0 -- SaaS Platform

**Status:** Complete
**Previous:** [v0.4.0](v0.4.0.md) | **Next:** [v0.6.0](v0.6.0.md)

## Summary

Transformed Studio UI from a local tool into a hosted SaaS product. Added Supabase-backed authentication, multi-user project management with role-based collaboration, an onboarding experience, a CLI scaffolding tool, and version history with diff viewing.

## Changelog

### Authentication (Supabase)

- [x] Email/password signup and login
- [x] GitHub and Google OAuth
- [x] User sessions verified via Next.js middleware
- [x] Public routes (`/login`, `/signup`, `/auth/callback`) bypass auth
- [x] Protected routes redirect to login when unauthenticated
- [x] Dual-mode: Supabase auth when env vars set, open access when not (local dev)

### Multi-User and Projects

- [x] Projects -- users have multiple projects, each with own screens/tokens/components
- [x] Persistence -- Supabase Postgres + Storage replaces filesystem in SaaS mode
- [x] Collaboration schema -- `project_members` table with viewer/editor/admin roles
- [x] Pending invites system -- invite by email, auto-converts to membership on signup
- [x] Share modal -- invite flow, member list, role management
- [x] RLS policies on all tables scoped to project membership

### Onboarding

- [x] Welcome modal on first visit
- [x] Interactive tutorial -- guided steps (drag Stack, add Heading, edit props, save, preview)
- [x] Template picker -- new users start from a template
- [x] Contextual tooltips on UI elements
- [x] Progress tracked in localStorage / user profile

### CLI Tool

- [x] `npx create-studio-app my-project`
- [x] Prompts: project name, framework (Next.js/Expo), include samples, include Supabase
- [x] Generates full project scaffold with config, tokens, sample screens

### Version History

- [x] Visual timeline of spec changes with diff view
- [x] JSON diff engine (flattenObject + computeDiff)
- [x] Colour-coded additions/removals/changes
- [x] Rollback to any previous version
- [x] Stored in Supabase (SaaS) or filesystem (local)

### Deployment

- [x] `vercel.json` for monorepo deployment
- [x] Auto-deploy from `main` branch
- [x] Root `/` redirects to `/studio` for authenticated users

### Packaging

- [x] SaaS (primary) -- hosted web app on Vercel
- [ ] Desktop (Tauri) -- deferred to v1.2.0

## Known Issues at Release

- RLS recursion between `projects` and `project_members` required `security definer` helper functions (migration 003)
- Asset storage still filesystem-based despite Supabase schema existing (planned for v0.8.0)
- Project creation uses bare text input, no modal (planned for v0.7.0)
- Signup page was serving a generated stub instead of real auth (fixed post-release)
