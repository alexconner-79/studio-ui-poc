# v0.7.0 -- Editor UX Overhaul

**Status:** Complete
**Previous:** [v0.6.0](v0.6.0.md) | **Next:** [v0.7.5](v0.7.5.md)

## Summary

A focused overhaul of the core editor experience. Addresses the most impactful UX gaps: project/screen/DS creation flows, canvas defaults, interactive spacing controls, component drop behaviour, breakpoint selector, and left menu organisation.

---

## Changelog

### 0.7.1 -- Project, Screen, and Design System Creation UX

- [x] **Project creation modal** -- replace inline text input with modal: name, description, framework picker (nextjs/vue/svelte/html)
- [x] **Screen creation modal** -- replace `window.prompt()` with modal: name input, optional template selector with visual cards, route prefix auto-suggestion
- [x] **Design System setup from main page** -- "Design System" button on screen list header (next to Import/Export), opens DS Wizard directly without entering editor first
- [x] **Delete project** -- context menu / action button on project cards, wired to `deleteProject` in queries.ts
- [x] **Delete screen** -- context menu / action button on screen cards, wired to backend
- [x] **Rename project and screen** -- inline rename or modal
- [x] **Duplicate screen** -- copies spec with a new name

**Key files:** `apps/web/app/studio/page.tsx`, `apps/web/components/studio/ds-wizard.tsx`, `apps/web/app/api/studio/projects/route.ts`, `apps/web/app/api/studio/screens/[name]/route.ts`

### 0.7.2 -- Canvas Defaults and Root Container

- [x] **Smart root container** -- when first component is dropped, auto-wrap in a vertical Stack (full-width) if not already a container type
- [x] **Root node styling** -- allow root node to be selected and styled (padding, background, gap)
- [x] **Better empty state** -- replace "Drag components here" with actionable options: "Start with a Stack", "Start from Template", or drag-and-drop area
- [x] **Default gap/padding** -- new screens have a root Stack with `gap: 1rem` and `padding: 1.5rem`

**Key files:** `apps/web/components/studio/editor-canvas.tsx`, `apps/web/components/studio/renderer.tsx`, `apps/web/app/api/studio/screens/route.ts`

### 0.7.3 -- Interactive Spacing Controls

- [x] **Three-zone canvas overlay** on selected nodes:
  - Outer zone (violet `#8b5cf6` at 15%) = margin. Hover shows label, drag adjusts margin per side.
  - Inner zone (teal `#06b6d4` at 15%) = padding. Hover shows label, drag adjusts padding per side.
  - Border zone (indigo `#6366f1`) = size. Drag handles to resize width/height.
- [x] **Design token snapping** -- drags snap to spacing token scale when DS is loaded (e.g., 4, 8, 12, 16, 24, 32, 48, 64); label shows token name + px value ("spacing-md (16px)"). Falls back to 8px grid when no DS active.
- [x] **Link-sides toggle** in property panel -- lock icon sets all 4 sides to same value
- [x] **Preset chips** -- "0", "xs", "sm", "md", "lg", "xl" quick-select above spacing inputs
- [x] **Visual box model diagram** -- Chrome DevTools-style diagram in property panel showing margin/padding/size, click to edit inline
- [x] **Batch spacing** -- "Set spacing for all elements" command palette action

**Key files:** `apps/web/components/studio/renderer.tsx`, `apps/web/components/studio/style-sections.tsx`, `apps/web/components/studio/style-fields.tsx`, `apps/web/lib/studio/store.ts`

### 0.7.4 -- Component Drop Behaviour

- [x] **Drop preview ghost** -- translucent placeholder at insertion point while dragging, shows component type label and approximate size
- [x] **Smarter drop targeting** -- hovering over non-container highlights nearest parent container; shows "will be added to [parent]" tooltip
- [x] **Post-drop selection** -- after dropping, auto-select the new component and scroll property panel to show its props
- [x] **Container validation hints** -- dim palette items that can't be dropped into the currently hovered container

**Key files:** `apps/web/components/studio/editor-layout.tsx`, `apps/web/components/studio/renderer.tsx`, `apps/web/components/studio/editor-canvas.tsx`

### 0.7.5 -- Breakpoint Selector

- [x] **Full-width option** -- "Full" breakpoint using 100% container width (no device frame)
- [x] **Active editing indicator** -- highlighted frame border on the breakpoint being edited; click a frame to switch
- [x] **Custom breakpoint** -- number input in breakpoint bar for arbitrary width
- [x] **Single-frame default** -- default to Desktop only; multi-frame is opt-in power-user feature

**Key files:** `apps/web/components/studio/editor-canvas.tsx`, `apps/web/components/studio/editor-layout.tsx`, `apps/web/lib/studio/store.ts`

- [x] **Component search** -- filter input at top of Components tab, searches across all categories including custom and templates
- [x] **Component favourites** -- star frequently-used components, starred items appear at top
- [x] **Screen search** -- text filter at top of screen list, filters by name or route

## Bug Fixes (post-release patch)

The following bugs were found during review and fixed in a follow-up patch:

- **Rename project does not save** — Added `PATCH /api/studio/projects` endpoint; rename input now saves on Enter and blur.
- **Spacing input: bare number not applied** — `SpacingEditor` now appends `px` when a bare integer is entered (e.g. `16` → `16px`).
- **"Full" breakpoint button has no visible effect** — `toggleFrame("full")` now exclusively sets the active frame to `["full"]` instead of appending to the list.
- **Color token picker empty in Supabase mode** — `loadDesignTokens` now passes `?projectId=...` so the tokens API returns the correct data. `currentProjectId` stored in Zustand, synced from `EditorLayout` on mount. **[Revisit before v1.0.0]** — tokens reload on every project change; verify there is no race condition on first mount that silently leaves the picker empty across different project configurations.
- **DragOverlay chip barely visible** — Redesigned with prominent purple border/glow for component drags, coloured variants for node-move, image, and composite drags.
- **shadcn Button/Input still blue** — Removed `--primary` and `--primary-foreground` overrides from `globals.css` `:root` and `.dark` blocks; `globals-studio.css` purple (#6c47ff) now takes effect. **[Revisit before v1.0.0]** — `globals.css` `:root` still contains many other shadcn token overrides that could conflict with future shadcn upgrades; consolidate all overrides into `globals-studio.css` as a single source of truth.
- **Token snap label not clearly visible** — `Pill` component now renders with a purple glow and `⚡` prefix when snapped to a token, making it instantly recognisable.

## Deferred Items

- **Multi-frame breakpoint editing (test 11 items 4–5)** — Clicking a frame to set `editingBreakpoint` and applying responsive overrides per-frame requires a dedicated UX planning session. Deferred to **v1.4.0 UX Refinement Pass**. Current behaviour: all edits apply globally regardless of which frame is active. **[Revisit before v1.0.0]** — global edits across all breakpoints is a correctness gap; users cannot make breakpoint-specific changes, which is a core feature claim of the product. Must be resolved before v1.0.0 ships.

## Known Limitations

- **Three-zone spacing controls on touch devices** — Permanent constraint. Pointer events are required; touch device support is out of scope for v1.0.0.
- **Token snapping only reads spacing tokens** — Addressed in v0.10.0 DS Authoring.
- **Component favourites stored per-browser** — Addressed in v0.11.0 Real-Time Collaboration.
- **Multi-frame per-breakpoint editing** — Addressed in v1.4.0 UX Refinement Pass. See BACKLOG.md — Multi-frame per-breakpoint editing.
- **Color token picker reliability in Supabase mode** — See BACKLOG.md — Color token picker reliability in Supabase mode.
- **shadcn CSS token cascade consolidation** — See BACKLOG.md — shadcn CSS token cascade consolidation.
