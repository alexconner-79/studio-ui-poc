# v0.1.0 -- Foundation

**Status:** Complete
**Previous:** -- | **Next:** [v0.2.0](v0.2.0.md)

## Summary

Stabilised the initial PoC and evolved it into a production-grade system. Established the spec format, multi-screen compiler, configuration system, and the browser-based visual editor that became the core of Studio UI.

## Changelog

### Hardening

- [x] E2E PR flow test -- `pnpm publish:pr` against a real GitHub repo with token auth, push, PR creation, duplicate detection
- [x] Error handling polish -- push failures, GitHub API rate limits, network errors with clear messages
- [x] CI validation -- GitHub Action runs `pnpm compile` on PR and verifies generated output matches committed files

### Spec & Compiler toward V1

- [x] V1 top-level structure -- `version`, `route`, `meta` fields; children under `tree` key
- [x] Node IDs -- every node gains an explicit `id` field
- [x] Props object -- structured `props` object replaces flat inline props
- [x] Screen discovery -- compiler discovers all `*.screen.json` files
- [x] Route-based emission -- each screen emits to its route path
- [x] Index generation -- auto-generated barrel exports for all generated components
- [x] `studio.config.json` -- defines framework, directories, schema path, import alias

### New Node Types

- [x] Layout primitives -- Grid, Section, ScrollArea, Spacer
- [x] Content nodes -- Image, Input, Link, Divider, List
- [x] Repo component references -- specs can reference any exported component

### Compiler Rules

- [x] Token-based spacing only -- enforced via validation
- [x] Self-closing elements -- `<Spacer />` etc.
- [x] Props type-checking -- spec props validated against TypeScript definitions

### Developer Experience

- [x] Visual preview -- watch mode with Next.js dev server
- [x] Spec scaffolding CLI -- `pnpm studio add screen checkout`
- [x] Colourised diff output -- syntax-highlighted diffs in `publish:dry`
- [x] Spec linter -- standalone lint for common spec issues

### Visual Editor & Long-Term Vision

- [x] Browser-based visual editor with drag-and-drop, property editing, undo/redo, save/compile
- [x] Multi-framework emitters -- Vue emitter alongside Next.js, abstract `Emitter` interface
- [x] Component library abstraction -- `ComponentAdapter` interface decouples from shadcn
- [x] Plugin / extension system -- `NodePlugin` interface for custom node types
- [x] Design token pipeline -- Style Dictionary JSON import, token-driven compiler maps

## Known Issues at Release

- Hardcoded paths in several API routes (tracked in TECH_DEBT.md)
- Full recompile on every save (no incremental compilation)
