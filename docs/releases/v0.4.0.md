# v0.4.0 -- Import / Export

**Status:** Complete
**Previous:** [v0.3.0](v0.3.0.md) | **Next:** [v0.5.0](v0.5.0.md)

## Summary

Opened Studio UI to the outside world. Users can bring in existing work from spec files, React codebases, and Figma designs. Finished projects can be exported as zip, deployed to Vercel, containerised with Docker, or pushed to GitHub. Added state management and data binding to bridge static mockups to functional prototypes.

## Changelog

### Spec File Import

- [x] "Import Screen" button on `/studio` screen list
- [x] Accepts `.screen.json` files (drag-and-drop or file picker)
- [x] Validates against schema, shows errors, loads into editor
- [x] Bulk import via zip

### Codebase Reverse-Engineering (Experimental)

- [x] AST parsing via `@babel/parser` + `@babel/traverse`
- [x] Maps HTML elements and shadcn components back to Studio node types
- [x] Extracts props, text, nesting structure
- [x] ~70-80% fidelity; user refines in editor
- [x] Marked as "experimental" in UI

### Figma Import

- [x] User pastes Figma file URL + provides Personal Access Token
- [x] API route fetches file JSON, runs parser, returns `ScreenSpec`
- [x] Mapping: FRAME -> Stack, TEXT -> Heading/Text, COMPONENT -> Card/Button, IMAGE -> Image
- [x] "Mapping review" step before committing
- [x] Layout properties (gap, padding) mapped to token values

### Project Export

- [x] Zip download -- bundles Next.js app as `.zip`
- [x] Vercel deploy -- one-click deploy via Vercel API
- [x] Docker export -- generates `Dockerfile` + `docker-compose.yml`
- [x] GitHub push -- extends `publish.ts` to create fresh repos

### State and Interactions

- [x] `onClick`, `onChange`, conditional visibility on nodes
- [x] Compiler emits event handler stubs
- [x] Bridges static mockup to functional prototype

### Data Binding

- [x] Connect components to API endpoints or mock data sources
- [x] List/DataTable nodes bind to data URL and auto-populate
- [x] Makes generated apps functional, not just visual

## Known Issues at Release

- Figma import captures ~20% of visual content (basic structure only) -- enhanced import planned for v1.3.0
- Codebase reverse-engineering marked experimental, edge cases exist
