# v0.10.10 — DS & Project Creation Flow

**Status:** Planned
**Depends on:** [v0.10.5](v0.10.5.md) (DS picker restructure must land first)
**Previous:** [v0.10.9](v0.10.9.md) | **Next:** [v0.10.11](v0.10.11.md)

## Summary

The relationship between projects and design systems is currently disjointed — a user creates a project, lands in the editor, and only discovers the DS connection exists later via Project Settings. This release fixes the flow so the two concepts are introduced together and the DS is always visible and meaningful in context. The goal is that after this release, no user should be confused about what a DS is or how it relates to their project.

> **Sequencing note:** This release is scheduled after the canvas design quality releases (v0.10.6–v0.10.8). The DS picker restructure in v0.10.5 changes which starters are surfaced and how they are framed. The project creation flow built in 0.10.10.1 and the DS workspace empty state in 0.10.10.4 must present the updated picker — leading with Studio boilerplate starter themes, not an undifferentiated library grid.

## Design Decisions

**Why a modal, not a page?** Creating a project is a lightweight action. A full-page wizard creates unnecessary navigation. The modal approach matches how Figma, Linear, and Notion handle object creation.

**Why inline DS creation in the modal?** The most common new-user scenario is "I have a new project and no DS yet." Forcing them to create the DS first, then come back, is a two-trip flow for a conceptually single action.

**Why not force DS selection?** "Skip for now" is retained because some users genuinely want to start laying out a screen before thinking about DS. The empty state guidance handles them on their own timeline.

- **Nested DS wizard UX complexity** — See BACKLOG.md — Quick DS setup abbreviated wizard. Implement the full wizard first; simplify based on user research.
- **"No DS" banner for existing projects** — Permanent constraint. The banner appears for all existing unlinked projects. Mitigated by making it dismissible per-project.
