# v0.9.5 -- Layout Intelligence

**Status:** Complete
**Previous:** [v0.9.0](v0.9.0.md) | **Next:** [v0.9.8](v0.9.8.md)

## Summary

Makes Build Mode layouts powerful enough that designers actively want to work in them. Introduces Figma-style auto-layout toggle on Frames, edge-pinning constraints, and three sizing modes (Fixed / Fill / Hug). This is the release that unifies free-form design and structured production layout into a single coherent model.

---

## Changelog

### 0.9.5.1 -- Auto Layout Toggle

- [x] Frames gain an "Auto Layout" toggle in the property panel Layout section (`style-sections.tsx`)
- [x] **Auto-layout ON** (default for new Frames):
  - Children flow via flexbox; direction toggle (Row / Column) updates `props.direction`
  - Gap, padding, justify-content, align-items controls appear when auto-layout is active
  - `Frame` added to `CONTAINER_TYPES` so Layout section is visible
- [x] **Auto-layout OFF**:
  - Layout section hides flow controls; only Overflow remains
  - Children can be freely dragged to absolute positions (v0.8.0 drag-to-move)
- [x] `convertToAutoLayout(frameId)` store action detects direction, computes gap, clears absolute positions from children
- [x] `breakAutoLayout(frameId)` snapshots visual positions and sets `position: absolute` on each child
- [x] Style type: added `flexDirection?: "row" | "column"` to `NodeStyle` for non-Frame containers

**Key files:** `apps/web/lib/studio/types.ts`, `compiler/types.ts`, `apps/web/lib/studio/store.ts`, `apps/web/components/studio/style-sections.tsx`

### 0.9.5.2 -- Constraints System

- [x] Added `constraints?: { horizontal?: ..., vertical?: ... }` to `NodeStyle` in both `types.ts` files
- [x] `ConstraintWidget` component exported from `spacing-overlay.tsx` -- shows pin buttons for horizontal and vertical anchors
- [x] Widget only visible when selected node's parent is a Frame with `autoLayout: false`
- [x] `ConstraintWidget` rendered in `property-panel.tsx` below the Style sections
- [x] CSS mapping implemented in `resolve-token.ts` (`resolvedStyleToCSS`) and `compiler/emitters/nextjs.ts` (`emitStyleAttr`)

**Key files:** `apps/web/lib/studio/types.ts`, `compiler/types.ts`, `apps/web/components/studio/style-sections.tsx`, `apps/web/components/studio/spacing-overlay.tsx`, `apps/web/components/studio/property-panel.tsx`, `compiler/emitters/nextjs.ts`

### 0.9.5.3 -- Sizing Modes

- [x] Added `widthMode?: "fixed" | "fill" | "hug"` and `heightMode?: "fixed" | "fill" | "hug"` to `NodeStyle`
- [x] `SizingModeField` component: mode dropdown + conditional value input/placeholder
- [x] `SizingSection` updated to use `SizingModeField` for both width and height
- [x] Renderer: `widthMode: "fill"` → `width: 100%`, `widthMode: "hug"` → `width: fit-content` (same for height)
- [x] Compiler: `widthMode`/`heightMode` handled before generic style loop, takes priority over raw `width`/`height`

**Key files:** `apps/web/components/studio/style-sections.tsx`, `apps/web/lib/studio/resolve-token.ts`, `compiler/emitters/nextjs.ts`

### 0.9.5.4 -- Responsive Sizing Behaviour

- [x] `widthMode`/`heightMode`/`constraints` are part of `NodeStyle`, so they are covered by the existing `responsive` override mechanism -- no additional wiring needed
- [x] Breakpoint-aware sizing mode dropdowns work because `SizingSection` receives the resolved `effectiveStyle` for the active breakpoint

**Key files:** `apps/web/lib/studio/types.ts`, `apps/web/components/studio/style-sections.tsx`

### 0.9.5.5 -- Layout Conversion Helpers

- [x] `detectLayoutDirection(children)` in `geometry.ts` -- compares horizontal vs vertical spread of children
- [x] `computeFlexGap(children, direction)` in `geometry.ts` -- averages spacing between children along flow axis
- [x] `snapshotAbsolutePositions(children, frame)` in `geometry.ts` -- reads DOM positions (falling back to style values)
- [x] `convertToAutoLayout(frameId)` store action -- uses helpers, sets `props.autoLayout = true`, clears absolute positioning from children
- [x] `breakAutoLayout(frameId)` store action -- uses snapshot helper, applies `position: absolute` with computed offsets, sets `props.autoLayout = false`
- [x] Context menu: "Add Auto Layout" / "Break Auto Layout" for Frame nodes (toggles based on current state)

**Key files:** `apps/web/lib/studio/store.ts`, `apps/web/lib/studio/geometry.ts`, `apps/web/components/studio/context-menu.tsx`

## Test Plan

### Prerequisites

- Dev server running
- v0.8.0 and v0.8.5 features complete (dual-mode canvas, shapes, drawing tools)
- At least one project with screens containing both Design Mode and Build Mode content

### Tests

**1. Auto Layout Toggle**
1. Create a Frame using the Frame tool
2. Verify auto-layout is ON by default
3. Drop three Button nodes into the frame -- verify they flow vertically with gap
4. Change direction to horizontal -- verify children reflow
5. Adjust gap and padding -- verify spacing updates on canvas
6. Toggle auto-layout OFF -- verify children jump to absolute positions
7. Toggle back ON -- verify the tool offers to re-sort by position

**2. Constraints**
1. Create a Frame with auto-layout OFF
2. Place a child Rectangle in the top-right corner
3. Set constraints: horizontal = "right", vertical = "top"
4. Resize the parent frame -- verify the child stays pinned to the top-right
5. Set constraints: horizontal = "left-right" -- resize parent, verify child stretches
6. Set constraints: horizontal = "center" -- verify child stays centred
7. Verify constraint widget on the selection bounding box updates correctly

**3. Sizing Modes**
1. Create an auto-layout Frame with three children
2. Set the first child to "Fill" width -- verify it stretches to fill remaining space
3. Set the second child to "Hug" width -- verify it shrinks to content size
4. Set the third child to "Fixed" 100px -- verify it stays at 100px
5. Compile -- verify output uses `flex: 1`, `width: fit-content`, and `width: 100px` respectively

**4. Responsive Sizing**
1. Set a node to "Fixed 300px" on Desktop
2. Switch to Tablet breakpoint, change to "Fill"
3. Switch to Mobile breakpoint, verify it inherited "Fill" from Tablet
4. Compile -- verify responsive CSS is emitted correctly
5. Preview at each breakpoint -- verify sizing changes

**5. Layout Conversion**
1. Create three absolutely-positioned nodes in a free-form Frame
2. Right-click the Frame > "Convert to Auto Layout"
3. Verify children are sorted by position and wrapped in flex layout
4. Verify gap is auto-calculated from the spacing between them
5. Right-click > "Break Auto Layout" -- verify children return to absolute positions matching their visual positions

**6. Stack/Box Backwards Compatibility**
1. Load an existing screen with Stack nodes
2. Verify Stacks render identically to before (they're Frames with auto-layout ON)
3. Open the property panel -- verify auto-layout toggle shows ON
4. Load a screen with Box nodes -- verify auto-layout toggle shows OFF
5. Compile -- verify output is identical to pre-v0.9.5

## Known Limitations

- **Constraints are 2D only** — Permanent constraint. Depth/z-index constraints are a separate concern; z-index is managed manually via the Position section.
- **"Scale" constraint mode pixel accuracy** — Addressed in v1.4.0 UX Refinement Pass.
- **Auto-layout conversion heuristic** — Addressed in v1.4.0 UX Refinement Pass. A UI toast notes the limitation at conversion time.
- **Sizing modes and constraints conflict** — Addressed in v1.4.0 UX Refinement Pass.
- **Expo emitter constraints** — Addressed in v0.10.7 Advanced Design via `FRAMEWORK_CONSTRAINTS`.
- **`snapshotAbsolutePositions` requires DOM** — Permanent constraint. Document positions are browser-only; SSR/compiler contexts fall back to stored style values.
