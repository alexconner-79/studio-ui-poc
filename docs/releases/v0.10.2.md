# v0.10.2 — Editor UX & Platform Preview

**Status:** Complete
**Previous:** [v0.10.1](v0.10.1.md) | **Next:** [v0.10.3](v0.10.3.md)

## Summary

A focused overhaul of the editor's core interaction model, navigation chrome, and platform awareness. Driven by a full product sense-check that identified five structural UX problems: disjoint project/DS creation, screens shown as cards instead of artboards, a cluttered Insert panel, incorrect cursor behaviour, and inability to select the artboard root. All five were addressed. On top of that, the editor gained framework-aware artboard sizing, an Expo-native preview (clean phone frame, no iframe), and DS-to-project style propagation fixes.

## Changelog

### 0.10.2.1 — Editor UX Sense Check

A systematic pass on the editor's core interaction model based on a full product review.

- [x] **Removed screens landing page** — projects drop directly into the design file; screen management lives in the left panel (Screens tab), not a separate card grid
- [x] **Cursor in selection mode** — changed from hand (`cursor: grab`) to arrow (`cursor: default`); hand cursor is reserved for panning only
- [x] **Deselect on canvas click** — clicking empty canvas space outside any node deselects the current selection; previously required pressing Escape
- [x] **Selection label no longer scales with zoom** — selection overlay labels (node name, artboard "Desktop" label) maintain a fixed visual size regardless of zoom level; slightly larger than previous default
- [x] **Removed dot-grid canvas background** — canvas background is now a flat dark colour; dot-grid was visually noisy and not adding value
- [x] **Icon rail widened to 50px** — the vertical icon rail to the left of the left panel increased from ~36px to 50px for better hit targets at 100% browser zoom
- [x] **Resizable left and right panels** — both panels have a draggable resize handle; current width is the minimum (panels cannot be made narrower than their default); behaviour matches Cursor's panel resize
- [x] **Layer naming — `name` field added to Node type** — nodes now carry a user-facing `name` field separate from the technical `id`; the Layers panel displays and edits `name`; `id` remains the internal reference and is never shown to users
- [x] **Artboard is selectable** — clicking the artboard background selects the artboard root node, allowing its properties to be inspected and edited in the right panel
- [x] **Insert panel restructured** — removed non-draggable structural items from the drag-and-drop list; items under "Structure" that apply to components (not draggable onto canvas) moved to the right panel
- [x] **Eliminated duplicate `next.config.ts`** — project had both `next.config.ts` and `next.config.js`; the `.ts` file was removed; only `next.config.js` remains (this was causing Next.js to hang at startup trying to install TypeScript transpilation dependencies)

**Key files:** `apps/web/components/studio/editor-canvas.tsx`, `apps/web/components/studio/editor-layout.tsx`, `apps/web/components/studio/node-tree.tsx`, `apps/web/lib/studio/types.ts`, `apps/web/next.config.js`, `spec/schema/screen.schema.json`

### 0.10.2.2 — Top Bar & Navigation

### 0.10.2.3 — Artboard Mode Split

The artboard size picker and editor canvas are now framework-aware. Web and Expo projects show different options and defaults.

- [x] **Framework-aware artboard presets**
  - Web (Next.js) projects: responsive breakpoints — Mobile 375px, Tablet 768px, Desktop 1280px, Custom
  - Expo (React Native) projects: named device sizes — iPhone 17 402px, Android Compact 412px, iPad 820px, Custom
- [x] **Expo default artboard width** — Expo projects default to 393px (iPhone 16 logical width) when the editor loads; web projects default to `null` (full-width responsive)
- [x] **Drag-to-resize artboard** — a drag handle on the right edge of the artboard allows manual width adjustment; delta is scale-corrected for the current zoom level; width written to `artboardWidth` in the Zustand store
- [x] **Device chrome removed from editor canvas** — the editor canvas shows a plain bordered artboard for all project types; phone/device outlines were removed from the canvas (they belong only in preview mode)
- [x] **Single artboard per screen** — multi-frame toggle (e.g. Mobile + Tablet + Desktop simultaneously) removed; replaced with a single active artboard size picker; multiple artboard sizes per screen are deferred to a future layout intelligence release
- [x] **`projectFramework` in store** — `artboardWidth: number | null` and `projectFramework: string` added to the Zustand store; both are set on project load from the API

**Key files:** `apps/web/components/studio/editor-layout.tsx`, `apps/web/components/studio/editor-canvas.tsx`, `apps/web/components/studio/device-frame.tsx`, `apps/web/lib/studio/store.ts`

### 0.10.2.4 — DS-to-Project Integration

Fixes to ensure design system tokens and styles actually propagate to the project editor and compiler.

- [x] **DS typography applies to project screens** — font family, size, weight, and line-height tokens from the linked DS are reflected in the canvas renderer and right panel
- [x] **Right panel reflects DS styles** — when a DS is linked, the property panel shows token-resolved values (e.g. `$color.primary → #2563eb`) alongside raw values
- [x] **Compiler always targets Next.js for in-app web compilation** — `apps/web/app/api/studio/compile/route.ts` now forces `framework: "nextjs"` regardless of `project.framework` in the database; previously, Expo projects were compiled with the RN emitter for the in-app web preview, producing React Native imports in generated `.tsx` files
- [x] **Tokens file absolute path** — the compile route resolves the tokens file path to an absolute path using `ROOT_DIR`; fixes the persistent "Tokens file not found" compile warning that appeared on every save
- [x] **Schema validation caching disabled in development** — `compiler/validate.ts` no longer caches the AJV validator instance in dev mode; fixes cases where schema changes were ignored until the server restarted

**Key files:** `apps/web/app/api/studio/compile/route.ts`, `compiler/validate.ts`, `apps/web/components/studio/renderer.tsx`, `apps/web/components/studio/style-sections.tsx`

### 0.10.2.5 — RN/Expo Preview (Clean Figma-Style Frame)

Web and Expo projects now have completely different preview experiences.

**Web projects** — unchanged: iframe-based preview with device chrome options (phone, tablet, desktop, responsive).

**Expo projects** — iframe removed entirely; spec tree is rendered directly in a clean phone frame:

- [x] **`PhoneFrame` component** — dark shell (`#1c1c1e`) with `52px` corner radius, physical side buttons (volume left ×2, power right), dynamic island (100×34px pill at top of screen), home indicator bar at bottom; no heavy hardware chrome or bezels; scales responsively to fit the browser viewport while maintaining exact device dimensions
- [x] **`NativePreviewRenderer` / `NativeNode`** — recursive renderer that walks the `ScreenSpec` node tree and outputs HTML+CSS; no Zustand store dependency, no DnD context; supports 16 node types: Stack, Grid, Section, ScrollArea, Spacer, Heading, Text, Button, Input, Card, Image, Divider, List, Nav, Tabs, Frame, Rectangle, Ellipse
- [x] **`NativeDevicePicker`** — mobile-only device dropdown for the Expo preview top bar; groups: iPhone (iPhone 17, iPhone 16, iPhone SE), Android (Compact, Normal), Tablet (iPad, iPad Pro 11″); selecting a device updates the `PhoneFrame` dimensions and scale
- [x] **Expo preview bypasses `hasCompiled` check** — preview button and `Cmd+P` shortcut do not require a prior compile step for Expo projects; the spec is fetched directly from the API and rendered natively
- [x] **`ExpoPreviewContent`** — top-level component for the Expo preview branch in `apps/web/app/preview/page.tsx`; fetches screen spec via `/api/studio/screens/[name]?projectId=[id]`, handles loading/error states, renders `PhoneFrame` + `NativePreviewRenderer` + `NativeDevicePicker`
- [x] **`framework` and `projectId` in preview URL** — editor passes both as query params; preview page branches on `framework === "expo"` to select the correct render path

**Key files:** `apps/web/app/preview/page.tsx`

## Known Limitations

- **`NativePreviewRenderer` is a best-effort visual approximation** — it renders the spec tree as HTML+CSS, not actual React Native. Layout behaviour may differ from a real device for complex flexbox or platform-specific styles. True device preview (via Expo Go or a simulator) requires a separate toolchain and is a future release item.
- **Drag-to-resize artboard minimum** — no minimum width is enforced on the drag handle; dragging to 0px collapses the artboard. A minimum of ~200px will be added in the next UX pass.
- **`name` field in schema** — `name` was added to `screen.schema.json` for all node types. Existing screen specs without `name` fields are valid (field is optional); the editor falls back to showing the node `id` in the Layers panel for legacy specs.
- **iCloud Drive compatibility** — this project is stored in iCloud Drive. macOS "Optimise Mac Storage" can evict `node_modules` files to iCloud, causing `next dev` to hang silently at startup. Fix: delete `node_modules` and run `pnpm install` to restore from the local pnpm store. Permanent fix: disable "Optimise Mac Storage" in System Settings → iCloud Drive.
