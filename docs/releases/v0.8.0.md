# v0.8.0 -- Dual-Mode Canvas + Direct Manipulation

**Status:** Complete
**Previous:** [v0.7.5](v0.7.5.md) | **Next:** [v0.8.5](v0.8.5.md)

## Summary

Introduces the **Path C dual-mode canvas** -- an explicit Design Mode (free-form, exploratory, non-compiling) and Build Mode (structured, auto-layout, compiles to clean code). Also makes the canvas interactive with resize handles, drag-to-move, smart guides, inline text editing, and multi-select actions.

**Philosophy:** Design freely, ship cleanly. The `compile` flag on every node guarantees that only production-ready content enters the compiler. Designers explore in Design Mode, then deliberately promote elements to Build Mode.

## Test Plan

### Prerequisites

- Dev server running
- At least one project with a screen containing several nodes (Stack, Heading, Text, Button, Image)

### Tests

**1. Mode Toggle**
1. Open a screen in the editor
2. Verify the Design/Build mode toggle is visible in the top bar
3. Click to switch to Design Mode -- verify the toggle updates
4. Press `D` -- verify it switches to Design Mode
5. Press `B` -- verify it switches to Build Mode

**2. Compile Flag**
1. In Design Mode, select a node
2. Right-click > "Move to Design" -- verify the node gets a dashed border and dimmed layer label
3. Switch to Build Mode -- verify the node renders at 80% opacity
4. Compile the screen -- verify the non-compile node is absent from the generated output
5. Right-click the node > "Promote to Build" -- verify it returns to normal rendering
6. Compile again -- verify it now appears in the output

**3. Resize Handles**
1. Select a node with explicit width/height
2. Drag a corner handle -- verify width and height update in the property panel
3. Shift+drag -- verify aspect ratio is maintained
4. Alt+drag -- verify resize happens from centre
5. Verify live dimension labels appear during drag

**4. Drag-to-Move**
1. In Design Mode, create a node with `position: absolute`
2. Drag the node on the canvas -- verify `top`/`left` update in the property panel
3. Switch to Build Mode -- verify dragging the same node reorders it in the tree instead
4. Select a flow-layout node in Design Mode -- verify dragging still reorders (not absolute move)

**5. Smart Guides**
1. Place two nodes side by side
2. Drag one node -- verify alignment guide lines appear when edges or centres align
3. Verify snapping: the node jumps to the guide position within 4px
4. Verify distance labels appear between snapped elements

**6. Inline Text Editing**
1. Double-click a Text node -- verify a text editing overlay appears
2. Type new text -- verify it appears in the overlay
3. Press Escape -- verify the text is committed and the overlay closes
4. Double-click a Heading node -- verify same behaviour
5. Verify the overlay matches the node's font, size, and colour

**7. Multi-Select Actions**
1. Shift+click to select three nodes
2. Verify a bounding box appears around all three
3. Click "Align Left" in the property panel -- verify all nodes align to the leftmost edge
4. Click "Distribute Vertically" -- verify equal spacing between nodes
5. Click "Group into Frame" -- verify nodes are wrapped in a Frame, visible in the layer tree

**8. Backwards Compatibility**
1. Load an existing screen (created before v0.8.0)
2. Verify all nodes default to `compile: true`
3. Verify existing drag-and-drop behaviour works in Build Mode
4. Compile -- verify output is identical to pre-v0.8.0

## Bug Fixes

- **R shortcut conflict**: Keyboard shortcut `R` was intercepted by a "rename selected node" handler before it could reach the Rectangle tool. Fixed by moving the rename action to `F2` and removing the early return that blocked all tool shortcuts starting with `r`/`R`.
- **Preview button UX confusion**: Two topbar buttons ("Preview" chip and external-link icon) looked similar and caused confusion. Fixed by renaming the chip to "Canvas Preview" (for in-canvas artboard preview) and the icon button to "Open Page" (for opening the compiled page in a new browser tab), with a visual divider between them.
- **Drawing tool coordinates wrong at non-100% zoom**: Shapes drawn with Rectangle, Ellipse, Line, and Frame tools were incorrectly sized/positioned at non-100% zoom because raw `getBoundingClientRect()` values (in viewport pixels) were being used directly as artboard coordinates. Fixed in `DrawingLayer` by dividing all draw coordinates by the current canvas `scale` (read from `CanvasControlsContext`), so drawn shapes are always correct regardless of zoom level.

## Known Limitations

- Drag-to-move only works for nodes with `position: absolute`; flow-layout nodes always reorder in the tree. [Planned: v0.8.9]
- Smart guides only snap to immediate siblings and parent, not to nodes in other subtrees. [Permanent constraint — see v0.8.9]
- Inline text editing does not support rich text (bold, italic) -- only plain text
- The `compile` flag is per-node, not per-subtree toggle (children inherit parent's compile skip but can't individually opt back in)
